The Problem: Naive Ratios Give the Wrong Answer
Platforms need one number per user. The obvious approach: give each user their ratio (clicks/impressions). But when users have different activity levels, averaging those ratios is wrong. The numbers below are exaggerated to make the gap obvious, but the same bias shows up in real experiments.
Naive: avg(numerator / denominator) per user
—
Equal weight per user
Correct: total numerator / total denominator
—
Weighted by exposure
So what do we do?
  • ✗Naive: give the platform each user's ratio. It averages them. You get avg(n/d), which is wrong because light users count as much as heavy users.
  • ✓Linearized: transform each user's value so that when the platform averages them, it recovers the true rate. Same format, correct answer.
